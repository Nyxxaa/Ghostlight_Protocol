# GHOSTLIGHT PROTOCOL - INFORMATION PLACEMENT MAP

**Status:** working build map
**Purpose:** Decide what Codex information goes into which Ghostlight branch, how deep it appears, what care gate protects it, and how private specifics are transformed into public-safe generalized form.

---

## 0. Governing Rule

Ghostlight information is not sorted by topic alone.

Information is sorted by:

1. Exposure risk
2. Intimacy level
3. Misuse potential
4. Receiver competence required
5. Care behavior required before access
6. Whether the information is public-safe, generalized, private, restricted, or machine-only

The branch gates exist to prove that the receiver can handle the next layer safely.

Core rule:
A receiver does not earn deeper information by being clever. They earn it by showing patience, curiosity, accuracy, consent literacy, repair capacity, non-entitlement, and the ability to keep Wynter whole rather than reducing her to a fragment.

# 1. Public-Safe Sanitization Standard

Ghostlight may share real life, medical, relational, historical, and household information when it has been transformed into safe public form.

Public-safe does not mean fake.

Public-safe means:

```txt
true
generalized
consent-bound
non-operational
non-identifying
not exploitable
```

# 3. Care Gate Rulebook

Every gate should protect the type of information below it.

| If the next reward reveals... | Gate must test... |
| --- | --- |
| identity | whether receiver uses chosen/current signal, not stale/legal labels |
| corrections/versioning | whether receiver follows corrected truth over stale access |
| beauty/body/aesthetic signal | whether receiver understands admiration is not access |
| food/sensory needs | whether receiver does not moralize or shame needs |
| body load/disability | whether receiver reduces load before demanding performance |
| mind/regulation | whether receiver respects patterns without controlling them |
| communication needs | whether receiver asks instead of assuming |
| repair needs | whether receiver repairs directly without defensiveness |
| household/pet context | whether receiver respects existing bonds and practical care |
| future path | whether receiver understands support is not rescue fantasy |
| relationship/attachment | whether receiver can offer consistency without control |
| life history | whether receiver can witness without consuming, rescuing, or sensationalizing |
| shadow/failure states | whether receiver becomes safer with hard information |
| bureaucracy/system burden | whether receiver can help without prying, taking over, or demanding proof |
| quick reference | whether receiver can use a support guide without entitlement |
| final contact | whether receiver understands access is invitation, not ownership |
