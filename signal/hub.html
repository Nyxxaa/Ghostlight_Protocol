optional local care-gate judge

Later care gates may use free-response answers. For the richer version, you can run a small local LLM on your own machine through LM Studio.

This setup is optional. The site will not install anything or run PowerShell for you. You must copy each command yourself and choose whether to run it.

Only do this on your own computer. Keep the LM Studio server bound to localhost. Do not expose it to your network or the public internet.

Step 1 — open PowerShell

Open PowerShell as your normal user, then copy and paste this command to install LM Studio with winget:

winget install --id LMStudio.LMStudio -e

If that package ID fails, run this search command and install the LM Studio result manually:

winget search "LM Studio"

Step 2 — open LM Studio once

After installation, open LM Studio from the Start Menu one time. This initializes the lms command-line tool.

Then return to PowerShell and verify that lms is available:

lms --help

Step 3 — download a small local model

Copy and paste this command to download a small instruction-following model:

lms get qwen2.5-3b-instruct@q4_k_m

If LM Studio cannot find that exact model, run this instead and choose a small Q4 instruction model from the results:

lms get qwen2.5-3b-instruct --always-show-all-results --always-show-download-options

Step 4 — load the model under the Ghostlight name

First list your downloaded model keys:

lms ls

Then load the model. If your model key is different, replace qwen2.5-3b-instruct with the exact key shown by lms ls.

lms load qwen2.5-3b-instruct --identifier ghostlight-care-judge --context-length 4096 --gpu auto

Step 5 — start the local server for browser care gates

This starts the LM Studio local server with browser access enabled. Keep it on localhost only.

lms server start --port 1234 --cors --bind 127.0.0.1

Step 6 — check that Ghostlight can see the model

Run this in PowerShell to confirm the model is loaded:

lms ps

Then use the relay test below.
